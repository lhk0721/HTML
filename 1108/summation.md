### Livingstandard
Living Standard 2019년 5월 28일부터 WHATWG와 W3C는 Living Standard를 HTML의 유일한 버전으로 발표했습니다. 이는 더 이상 HTML5와 같은 버전 번호를 사용하지 않고, HTML 표준을 지속적으로 업데이트하고 발전시키는 방향으로 나아가기로 합의한 것을 의미합니다.

### HTML 기본 구조
* <!DOCTYPE html> : HTML living standard로 작성된 문서입니다.
  - Document type Definition
  -  DOCTYPE을 제거하면 브라우저는 오래된 웹 페이지로 간주하고, 쿼크 모드(quirks mode : 하위 호환성 모드)로 html을 렌더링하게 됩니다. 그렇기에 DOCTYPE을 반드시 문서 최상단에 선언
  
* <html>
     - 모든 HTML 요소는 <html> 태그 내에 포함되어야 합니다
     - lang 속성을 통해 해당 페이지의 주 언어가 무엇인지 설정할 수 있습니다. 검색엔진, 스크린리더, 번역 기능 제공 등에 영향을 미치게 됩니다.
     - 언어코드(소문자)-국가코드(대문자)
      | 언어코드 | 언어   | 국가코드(국가)                              |
      |-----------|--------|--------------------------------------------|
      | ko        | 한국어 | KR(대한민국), KP(북한)                    |
      | en        | 영어   | US(미국), GB(영국), PH(필리핀)            |
      | zh        | 중국어 | CN(중국), HK(홍콩), TW(대만), Hans(간체), Hant(번체) |
      | ja        | 일본어 | -                                          |
      | de        | 독일어 | -                                          |

* <head></head>
    - 문자 인코딩, 타이틀, 설명, 스타일시트, 스크립트 등 기계가 식별할 수 있는 문서 정보(메타데이터)를 담습니다.
    - 사용자에게는 제목(title), 파비콘(favicon), 뷰포트(viewport) 정보 등이 보이게 됩니다.
* <meta>
    - 메타데이터: “어떤 목적을 위해 만들어진 데이터”
    1. charset : 문자 코드의 종류를 설정
    2. <meta http-equiv="X-UA-Compatible" content="IE=edge" />
      - 한 태그에 여러 name/content 속성을 섞는 건 피해야 하고,메타데이터 하나당 <meta> 하나씩 나누는 게 정석입니다.
        http-equiv
          - “HTTP 응답 헤더처럼 작동하라”는 의미의 속성입니다.
          즉, HTML 내부에서 HTTP 헤더 역할을 흉내 내는 용도입니다.

        "X-UA-Compatible"
          - “User Agent 호환성 모드”를 뜻합니다.
          IE에게 호환성 보기 모드(오래된 엔진으로 렌더링하는 기능)를 끄고
          최신 엔진(edge)으로 렌더링하라고 지시합니다.

        content="IE=edge"
          - IE가 가능한 가장 최신 버전의 렌더링 엔진(EdgeHTML, 또는 Chromium 기반 Edge라면 Blink)을 사용하도록 설정.

        브라우저 호환성을 위해 작성해줍니다. IE의 최신 표준 모드를 보여주겠다는 의미입니다.
        IE의 다양한 버전(6-11) 때문에 같은 페이지도 다르게 보여지는 경우가 있었습니다. 그러한 문제를 해결하기 위해 작성해주는 코드입니다
    - name=”” content=””
        name과 content 특성을 함께 사용하면 문서의 메타데이터를 이름-값 쌍으로 제공합니다.
        name은 화면 전체 페이지에 적용되는 "문서 레벨 메타데이터”를 제공합니다. 어떤 형태의 정보(viewport, description, author…)를 가졌는지를 의미합니다.
        content는 실제 메타데이터의 요소입니다.
    - viewport
        이 코드는 모바일 장치와 같은 다양한 디바이스에서 웹 페이지의 표시 방식을 제어
        - width: 뷰포트의 너비를 제어합니다. width=600 과 같이 픽셀값으로 지정할 수도 있습니다.
        - height: 뷰포트의 높이를 제어합니다.
        - initial-scale : 페이지가 처음 로드될 때 확대/축소 수준을 제어합니다. 
        - 최소 0.1 - 기본 1 - 최대 10
        - minimum-scale: 축소 정도를 제어합니다.
        - maximum-scale: 페이지에 허용되는 확대 정도를 제어합니다. 3보다 작은 값은 접근성에 맞지 않습니다
        - user-scalable=no 과 속성을 주어 사용자가 화면크기를 제어 못하게 하는 경우도 있습니다. 하지만 다양한 디바이스 크기, 노안, 저시력 등의 다양한 상황을 고려해야 하므로 특수한 상황이
        아니라면 사용하지 않는 게 좋습니다. 해당 속성을 사용해도 접근성을 위해서 무시하는 브라우저도 있습니다.
* <title></title>
    - 현재 문서와 외부 리소스의 관계를 명시
    - 검색엔진은 이 제목을 보고 사용자가 탐색할 페이지를 선택합니다
    - 특수문자는 제외하고 텍스트만 포함, 약 60자를 넘지 않아야 합니다.
* <link>
    <link rel="stylesheet" href="./css/style.css"> 빈 태그, 속성만을 포함, head 요소 내부에만 위치
    - 현재 문서와 외부 리소스의 관계를 명시합니다.
    - 외부 스타일 시트, 폰트, 파비콘을 연결할 때 사용합니다.
    - rel: relations 관계. 대상 파일의 속성을 나타냅니다
    - href: hyper-references 경로. 연결 시 참조할 파일의 위치를 나타냅니다.
    - 파비콘은 rel을 <link rel="icon"> 또는 <link rel="shortcut icon"> 형식으로 선언해야 인식됩니다.
    ```html
    <link rel="icon" type="image/png" href="images/favicon.png">

    ```
* <body></body>
    -body 태그는 사용자에게 보이는 영역

### HTML 태그
* 태그(Tag)
    - 어떤 표시를 하기 위해 붙인 꼬리표. 웹 문서에 정보를 정의해주는 형식
    - <> 시작 태그(여는 태그) </> 종료 태그(닫는 태그)의 쌍
    - 여는태그 | 컨텐츠 | 닫는태그
    - 요소 안에 다른 요소가 들어갈 수 있음. 단, 상위 요소 안에서 닫혀야 한다.
* 빈 요소
    - 태그가 없는 요소입니다. <br>, <img>, <input> 등이 있습니다.
* 주석 
    - ctl + / : 주석처리 단축키
    - 프로그래밍 소스를 이해하기 쉽게 메모
        - 태그의 시작과 끝을 적어놓기
        ```html
        <!-- start -->
        <!-- //end -->
        
        <!-- header -->
        <header>...생략...</header>
        <!-- //header -->
        
        <!-- footer-->
        <footer>...생략...</footer>
        <!-- //footer -->
        ```
        - 협업할 때
        ```html
        <!-- 메뉴 토글 class="active" 유/무로 제어 -->
        <nav class="active">...생략...</nav>
        
        <!-- #2022.02.20 4:00 업데이트 -->
        ```
        - 임시 처리
        ```html
        <!-- 신청기간이 아니여서 임시 주석처리
            <a href="#">신청페이지 바로가기</a>
        -->
        ```
    - 특정 소스를 비활성화 하는 것 ("주석처리 한다" 라고 이야기합니다.)
    - 매우 중요한 기밀 사항을 적어놓는 것은 피해야 합니다.

### <h> 제목태그
* 숫자가 작을수록 크기가 큰 제목을 나타냅니다
* 제목 태그는 검색엔진 사이트나 스크린리더를 사용하는 사용자에게도 매우 중요한 정보로 활용되므로,  페이지의 접근성과 검색 엔진 최적화에 있어서 필수적입니다.
* <H1></H1>
  - **h1**은 페이지의 주요 제목으로, 한 페이지에 하나를 사용하는 것을 권장
  - SEO와 스크린리더 모두에게 문서 구조를 전달하는 핵심 시맨틱 요소
  - 시각적 크기 조절은 CSS로 처리하고, 제목 태그는 의미 위주로 사용합니다.
  - h1 ~ h6 순서대로 작성해야 합니다 중간 단계를 생략하면 안됨.
    ```html
      <!-- O -->
      <h1>위니브</h1>  <!-- 문서 전체 주제 -->
        <h2>위니브 캐릭터</h2>
          <h3>로지</h3>
          <h3>개리</h3>
        <h2>위니브 소개</h2>
          <h3>위니브는...</h3>
      <!-- O -->
      <h1>위니브</h1>
      <h3>라이캣</h3>
      <!-- X -->
      <h2>위니브 캐릭터</h2>
      <h3>로지</h3>
      <h3>개리</h3>
      <h2>위니브 소개</h2>
      <h4>위니브는...</h4>
      <!-- X -->
      <h3>위니브 커리큘럼</h3>
    ```
* <a>
  - 하이퍼링크를 생성하는 데 사용되는 기본 요소
  - href 속성을 통해 연결할 URL
  - 사용자가 클릭하면 지정된 위치로 이동하여 웹페이지 간 이동을 가능하게 합니다.

  * 연결방식
    - 기본 연결
      1. 절대경로
      ```
        href="https://www.example.com"
      ```
      2. 상대경로
      ```
        href="/about"
      ```-
      3. 같은 페이지의 요소로 연결
      ```
        href="#section1"
      ```

    - 프로토콜 기반 연결
      1. tel: 전화번호: 클릭 시 전화 앱이 열리도록 합니다. 예: 
      ```
      href="tel:+1234567890"
      ```
      2. mailto: 이메일주소: 클릭 시 이메일 클라이언트가 열리도록 합니다. 예: 
      ```
      href="mailto:example@example.com"
      ```

  *속성값
    - target 속성값
      1. _self: 링크를 현재 페이지에서 엽니다. (기본값)
      2. _blank: 링크를 새 탭에서 엽니다.
      3. _parent: 부모 프레임에서 엽니다.
      4. _top: 최상위 창에서 엽니다.
      5, 프레임명: 지정된 프레임이나 창에서 엽니다.

    - download 속성
    download 속성을 사용하면 링크를 클릭할 때 페이지 이동 대신 파일 다운로드를 수행합니다. 
    ```
    <a href="document.pdf" download="새파일명.pdf">PDF 다운로드</a>
    ```

* <code>
  - 짧은 코드 조각이나 한 줄의 코드를 나타낼 때 사용
  - 텍스트가 고정 폭 글꼴로 표시되어, 코드나 명령어가 다른 텍스트와 구별되도록 합니다.

* 사용예시
 - pre가 outerspace에 있어야 한다. 
```
  <pre>
    <code>
    <!-- write code here -->
    </code>
  </pre>
```

* Entity Code
  - 특수문자를 표현하기 위한 코드.
  - 특수문자 입력이 코드 오류로 표현되지 않게 엔티티코드로 수정해서 사용.
    1. 공백 : &nbsp;
    2. 꺽쇠 : &lt; , &gt;
    3. 앰퍼샌드 : &amp;
    4. 큰 따옴표 : &quot;
    5. 작은 따옴표 : &apos;
    6. 카피라이트 : &copy;

* <strong>
  - 중요한 내용, 경고, 핵심 정보를 나타낼 때 사용
  - 핵심은 시각 효과가 아니라 의미(importance)
  - 몇몇 스크린 리더는 <strong>을 만나면 보통 억양을 달리하거나 강조

* <em>
  - 강조(emphasis) 를 표현, 강조보다는 강세
  - 기울임꼴(italic)로 표시되지만, 중요한 것은 억양·강조의 의미 변화
  - 중첩 사용이 가능하며, 중첩될수록 강조의 정도가 더 강해지는 의미를 가집니다.

* <i>
  - 문장 흐름과 다른 성격의 텍스트를 나타낼 때 사용
  - 외국어, 용어, 생각, 기술 용어, 인용 속 등장인물의 말투, 아이콘 글리프 등
  - <em>처럼 “강조”의 의미는 아니고, “성격이 다른 텍스트”를 표시하는 용도

* <address>
  - 콘텐츠에 대한 연락처 정보
  - 물리적 주소, URL, 이메일 주소, 전화번호, SNS 정보, 좌표 등을 포함
  - 개인, 조직, 단체의 이름 명시: 연락처 정보를 제공하는 개인, 조직, 또는 단체의 이름을 반드시 적어야 함.
  - 페이지 헤더나 푸터에 배치하여 사업체의 연락 방법으로 사용할 수 있습니다. 또한, article 내에 사용하여 글의 작성자를 나타낼 수도 있습니다.
  - 연락처 외의 정보(예: 출판일 등)를 포함해서는 안 됩니다.

    - start 속성을 통해 시작 번호 지정
    - rebversed 속성을 통해 순서를 거꾸로 시작

      <ol>
        <li>1번</li>
        <li>2번</li>
        <li>3번</li>
      </ol>
    ```
    ```html
      <ol type = >
        <li>1번</li>
        <li>2번</li>
        <li>3번</li>
      </ol>
    ```

  * <dl>
    - Definition List(설명 목록)의 줄임말로, 용어와 그 설명을 짝지어 표현하는 특별한 목록
    -  용어와 그 설명(정의)을 짝지어 표현할 때 사용, 키-값 구조(key-value pair) 의 정보를 표현하는 데 적합
    - 회사 소속원 목록 등에 사용
    - <dl>의 자식 요소로 <dt>, <dd> 이외 다른 태그가 올 수 있다. 
      - <dl>:전체 설명 목록을 감싸는 컨테이너 입니다.
      - <dt>: 정의되는 용어(term)를 나타냅니다.
      - <dd>: 용어의 설명(description)을 나타냅니다.
  ```html
  <dl>
    <dt>서울</dt>
    <dd>대한민국의 도시 입니다.</dd>
    <dt>평양</dt>
    <dd>북한의 도시 입니다.</dd>
  </dl>
  ```

  * 블록 요소와 인라인 요소
    - <p>, <div>는 블록 요소
    - <span>, <a>, <em>, <strong>는 인라인 요소
    - 블록 요소 내에 인라인 요소를 넣을 수 있지만, 역은 성립하지 않는다. - - (a)태그는 인라인 요소지만 중첩이 가능한 특이한 특징을 가진다.
    - 블록 요소의 특징: 한 줄을 전체 차지, 이외 요소는 줄바꿈된다.
    - 인라인 요소의 특징 : 블록 레벨 요소 내에 포함, 컨텐츠에 따라 할당된 공간만 차지. 새로운 줄을 만들지 않습니다.
    1. <div>
      - division의 약자. 
      -  웹 페이지의 레이아웃을 만들기 위해 콘텐츠를 분할하는 블록 레벨 컨테이너
      - 아무런 목적 없이 레이아웃을 나눠주는 태그.
      - 검색엔진 최적화, 코드 가독성, 접근성 등의 이유로 div 태그 사용은 스타일 적용을 위한 용도로 사용할것을 권장
      - header, footer, main, section, article, nav 등 다양한 의미가 있는(시멘틱한) 태그들을 사용하여 콘텐츠를 분할하고 그룹핑
      
    2. <span>
      - 인라인 컨테이너로, 스타일을 적용하거나 인라인 요소를 묶는 데 사용
      - 본질적으로는 아무런 의미를 나타내지 않지만, 특정 텍스트나 인라인 요소에 CSS 스타일을 적용할 때 유용
      - 문단 내에서 특정 글만 스타일을 주고 싶을 경우, span 태그로 묶는다.
      
    3.  inline-block 요소

* <img> 이미지 태그
  - HTML 문서에 이미지를 삽입하는 데 사용
  - 검색엔진은 이미지 파일명을 중요한 요소로 활용하기 때문에, 명확한 파일 이름을 사용.
  - src 속성을 통해 이미지 경로 지정
  - alt 속성을 통해 이미지에 대한 대체 설명 넣기. 넣을 말 없어도 빈 값으로 넣음. 
  - title 속성을 통해 마우스 호버 했을 때의 설명을 넣음. 
  - 추가 설명이 필요할 경우 <figure>와 <figcaption>을 사용.
  ```html
  <img  src="images\penguin.jpg"/>
  ```
