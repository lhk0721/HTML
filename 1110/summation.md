### 실습 코드 리뷰
* <dl>
    - P로 불필요하게 묶을 필요 없다.
    - 부연설명이 아닌 정의(discription) K : value 
* 제목태그
    - 제목을 적는다. 스크린 리더가 읽을 것을 생각하자.
* 이미지 태그
    - 이미지에 대한 설명을 넣는다.
    - '이미지'라는 단어는 중복되지 않기 위해 생략한다.
* 역슬래시
    - 링크는 슬래시로 넣는다.
    - 역슬래시(\\)는 HTML 문법에서 특별한 의미가 없고, 잘못된 경로나 태그 오류를 일으킬 수 있다.

### 이미지태그~
* <img>
    - alt 속성을 잘 써주는 게 중요하다.
        - 단순 꾸미기 위한 것이고, 단순히 장식용이면 alt를 비우면 된다.
        - 단, alt 속성 자체를 생략해선 안된다.
    - src 속성은 역슬래시가 아닌 슬래시를 사용한다.
    - 이미지는 단일태그다. <source> 쓸 수 없다.

* <figure>, <figcaption>
    - 문서의 주요 흐름과 독립적으로 참조될 수 있는 자체 완결적인 콘텐츠
    - <figcaption>요소는 이 콘텐츠에 대한 설명(캡션) 을 제공
    - 이미지뿐만 아니라 코드 조각, 인용문 등에도 사용 가능합니다.
    -  문서의 주요 내용에서 참조하는 일러스트레이션, 다이어그램, 사진, 코드 목록 등을 그룹화하는 데 사용
    - 이미지, 사진, 도표, 등에 추가적인 설명이 필요할 때 사용, 스크린리더가 읽어줌.

### video & audio
* <video>
    - 플래시를 대체하여 영상을 삽입할 수 있는 video 태그가 탄생한 후, 현재는 비디오 태그를 사용하여 동영상을 재생
    - src : 동영상 주소. 로컬 파일만 설정할 수 있다.
    - 'controls' : (값 없는 단일 속성). 컨트롤바 
        - 컨트롤바는 꼭 넣어보자
    - 'autoplay' : 소리가 있는 동영상의 경우 브라우져에서 자동재생을 차단.
    - 'muted' : 음소거 자동재생
    - 'loop' : 무한반복
    - 'poster' : 썸네일
    - preload : 어떻게 미리 로드할 지 제어
        a. none : 비디오 파일을 미리 로딩하지 않습니다. 서버가 최소한의 트래픽을 유지하며 페이지 로딩이 좀 더 빨라집니다. -> 페이지에 여러가지 비디오가 있을 때, 광고일 때
        b. metadata : 비디오 파일을 미리 로딩하지 않지만 파일의 메타데이터(길이 등)를 미리 가져오도록 합니다.-> 비디오의 정보만 주고, 재생할지 말지는 사용자 결정에 맡길 때
        c. auto(기본값) : 비디오 파일을 미리 로딩하여 사용자가 바로 영상을 볼 수 있도록 준비합니다. -> 즉각 사용이 기대될 때. delay 없이 재생이 되어야 할 때
    - <track> : <audio> 혹은 <video> 요소의 자식으로 자막과 같은 시간 기반 텍스트 데이터(텍스트 트랙)를 보여주고자 할 때 사용
    - 단일 태그만 쓰면 호환성 문제가 있다.

* <source>
    - <video> 또는 <audio> 태그 내에서 여러 미디어 파일을 지정
    크로스 브라우징
        - 다양한 브라우저 간의 호환성을 확보하여, 사용자가 어떤 브라우저를 사용하더라도 일관된 사용자 경험을 제공하기 위해 중요
        - 여러가지 형식의 비디오를 준비해 사용자 호환성 문제 예방.
    - type 속성이 없으면 브라우져가 별도로 호환성 검사하느라 자원을 낭비하므로 꼭 써주기.

*<audio>
    - 웹 페이지에 오디오 컨텐츠를 포함할 때 사용
    - 지원하는 형식
        1. MP3
        2. WAV
        3. Ogg
    - src :  경로 명시
    - <source> 요소를 자식으로 사용할 수 있습니다.
        - type 속성을 꼭 쓰기!!
    - 'autoplay' : 자동 재생 (쓰지 않기.)
    - loop
    - preload

### iframe

* <iframe>
    - inline frame 의 줄임말.
    - 유튜브 영상, 음원, 구글지도 등을 표시해주고 싶을 때 사용하나, 보안문제로 거부당하기도 한다. 
    - 악성 사이트들의 클릭재킹 문제도 있다.
    - src : 삽일될 문서의 경로
    - frameborder는 CSS에서 관리하므로 지워버린다.
    - width: 너비 지정(px,%)
    - height: 높이 지정(px,%)
    - scrolling: 스크롤바 표시 유무(yes,no,auto)
    - longdesc: 내용을 설명하는 페이지
    - marginwidth: left(좌), right(우), 여백공간(margin)
    - align: 정렬(top, middle, bottom, left, right)
    - allow : iframe 에서 허용할 기능들을 지정합니다.(카메라, 마이크 등)

* 유튜브 iframe
    - 공유 -> 퍼가기 -> iframe 코드 복사 

* 구글지도 iframe
    - 공유 -> 지도 퍼가기로 코드 복사 가능

### SVG, Canvas
* Canvas는 주된 태그는 아니라서 생략
* svg 형식
    - 확장 가능한 벡터 그래픽(scalable vector graphics)으로 XML 기반의 2차원 그래픽. 픽셀이 아닌 수학적 경로와 도형으로 이미지를 표현하므로 어떤 크기로 확대해도 품질이 유지
    - 코드 집합으로 이루어져있음.
    - 자바스크립트로 조종 가능.
    - 확장성: 어떤 해상도에서도 선명함.
    - 편집 가능: CSS로 색상, 크기 변경 가능
    - 접근성: 텍스트 정보 포함 가능 (alt 텍스트 등)
    - SEO 친화적: 검색엔진이 내용을 읽을 수 있음
    - 작은 파일 크기: 단순한 도형이나 아이콘의 경우 매우 작음
    - 복잡한 이미지의 경우 파일사이즈가 커짐
    - 사용방법
    1. img 태그 src 속성으로 넣기.
        - 구현 간단. 재사용 관리, 캐싱 가능.
        - javsscript로 관리 어려움, 내부요소 조작 제한적, css스타일링 불가.
    2. CSS background 배경으로 넣기(CSS 배우고 다시 공부)
        - 교안 참조.
    3. 인라인으로 넣기.
        - svg 코드 자체를 삽입하는 방법.
        - css, javascript로 관리 용이.
        - 추가 http 요청이 없어 로딩속도 향상.
        - 동적으로 svg 내용 변경 가능.
        - 가독성 크게 해침. 캐싱 불가능, html 파일 무거워짐.

### 그 외 다양한 태그들 (비교적 최신 태그들)
* 필기 이외 태그들 교안 읽어보기

* <dialog>
    - 대화 상자나 모달 창(modal window) 같은 대화형 컴포넌트를 만드는 데 사용
    - 페이지 위에 레이어를 까는 개념. 
    - JS 배우고 다시 공부..!

* <details> , <summary>
    - 토글의 UI.

* <picture>
    - 장치, 환경에 따라 각각 다른 포맷, 유형의 이미지 보여줌.
    - <source>(후보, 경로는 sourceset 사용), <img>(필수, fallback 이미지)랑 같이 쓰임.
    - sourceset으로 경로 지정하면 여러 후보 지정 가능
    - 사용자가 어떤 환경이냐에 따라 조정해 속도관리를 용이하게 할 수 있다.

* <time>
    - 특정 시간 또는 날짜를 시멘틱하게 나타내는 데 사용
    - 브라우져가 읽을 수 있는 시간 표시.
